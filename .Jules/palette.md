## 2024-05-17 - Accessible Form Submissions and Validation Feedback
**Learning:** During form submission, button disabled states must be explicitly tied to both `loading` and `form.invalid` states. Furthermore, simply showing a loading message outside the button is poor UX; the loading state (e.g. 'Processing...') should be placed inside the button to prevent duplicate submissions and provide direct feedback. All dynamic form validation messages must use `role="alert"` so screen readers proactively announce the errors. Form fields without explicit `<label>` tags (using `id` and `for`) must have `aria-label` attributes to ensure accessibility.
**Action:** When creating or modifying forms, always ensure the submit button handles `loading` and `invalid` states internally. Wrap validation messages with `role="alert"` and ensure all inputs have `aria-label` or explicit labels.
