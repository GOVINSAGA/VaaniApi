## 2024-05-18 - Missing Explicit Labels and Alert Roles on Auth Forms
**Learning:** Found that basic authentication forms relied solely on placeholders for input identification and standard divs for error messages. Screen readers require explicit `<label>` tags linked to inputs via `for`/`id` pairs to announce fields correctly, and error messages need `role="alert"` to be immediately announced when validation fails. Placeholders are also inadequate for visible labels as they disappear once the user types.
**Action:** When adding or reviewing forms, always verify that every `<input>` has a visible, explicitly associated `<label>` (using `for`/`id`) and that any dynamic error/validation messages use `role="alert"` to ensure they are announced by screen readers promptly. Also ensure buttons provide visual and programmatic feedback during async operations.
